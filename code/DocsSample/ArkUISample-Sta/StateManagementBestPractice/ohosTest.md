# 状态管理优秀实践测试用例归档

## 用例表

| 测试功能 | 预置条件 | 输入 | 预期输出 | 是否自动 | 测试结果 |
|---------|---------|------|---------|:-------:|:-------:|
| 不使用状态变量强行更新(反例) | 设备正常运行 | 点击add item按钮 | 数组添加项并触发UI更新 | 是 | Pass |
| 使用@Local装饰状态变量(正例) | 设备正常运行 | 点击add item按钮 | 数组添加项并自动触发UI更新 | 是 | Pass |
| 精准控制关联组件数(反例) | 设备正常运行 | 点击move按钮 | 多个组件同时刷新 | 是 | Pass |
| 精准控制关联组件数(正例) | 设备正常运行 | 点击move按钮 | 仅父组件刷新 | 是 | Pass |
| 对象类型冗余刷新(反例) | 设备正常运行 | 点击Change prop1按钮 | prop1和prop2关联组件均刷新 | 是 | Pass |
| @ObservedV2/@Trace精准刷新(正例) | 设备正常运行 | 点击Change prop1按钮 | 仅prop1关联组件刷新 | 是 | Pass |
| 循环频繁读取状态变量(反例) | 设备正常运行 | 点击print logmessage按钮 | 循环中每次读取状态变量 | 是 | Pass |
| 预先读取到局部变量(正例) | 设备正常运行 | 点击print logmessage按钮 | 循环外读取一次到局部变量 | 是 | Pass |
| 直接操作状态变量(反例) | 设备正常运行 | 点击change按钮 | 多次触发状态变量更新 | 是 | Pass |
| 使用临时变量(正例) | 设备正常运行 | 点击change按钮 | 临时变量计算后一次性赋值 | 是 | Pass |
