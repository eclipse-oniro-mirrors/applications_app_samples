# 视频播放器测试说明

## 1. 概述

本说明用于指导 `TVDemandPlayer`（点播播放器）示例应用的功能验证与测试操作，并结合自动化测试用例（位于 `entry/src/ohosTest/ets/test/`）给出可执行的验证清单。

该示例面向 OpenHarmony TV/大屏场景，基于 Stage 模型实现：

- 网络点播：进入网络视频列表并播放在线媒体资源
- 本地视频：按目录浏览本地媒体文件并播放
- 文件管理器拉起播放：通过 `ohos.want.action.viewData` 直接拉起播放器播放指定文件
- DLNA 播放与分布式流转：在支持的设备/系统配置下进行投屏、流转等（依赖环境）

> 说明：分布式流转、DLNA、后台拉起等能力对设备形态、系统能力、权限与网络环境要求较高；若环境不满足，相关功能可能无法验证。

## 2. 测试环境

### 2.1 设备与系统

- 设备类型：TV/大屏设备（支持遥控器或键盘操作）
- 系统版本：建议使用与工程配置匹配的 SDK 版本（工程 `compileSdkVersion`/`compatibleSdkVersion` 为 `18`）

### 2.2 网络

- 网络点播、DLNA、分布式流转等功能需要设备处于可用网络环境

### 2.3 构建与安装

- 使用 DevEco Studio 打开工程 `code/SystemFeature/TV/TVDemandPlayer`
- 选择目标设备后编译并安装

## 3. 自动化测试用例说明（ohosTest）

本工程在 `entry/src/ohosTest/ets/test/` 下提供了基于 `@ohos/hypium` 的测试用例，命名通常为：

建议在补充/回归测试时优先参考对应的 `.test.ets` 文件，确保手工验证点与自动化契约一致。

### 3.1 UI 冒烟场景（页面可达性与 Hint 校验）

对应文件：`entry/src/ohosTest/ets/test/UiSmokeScenario.test.ets`

该文件维护了 UI 冒烟场景清单，覆盖典型路径与页面提示文本（Hint）的存在性校验：

- 主页 → 网络视频：目标页面 `pages/HttpVideoList`，Hint 包含 `网络点播` / `暂无网络视频`
- 主页 → 本地视频：目标页面 `pages/LocalFolderList`，Hint 包含 `本地点播` / `本地视频`
- 网络列表 → 播放器：目标页面 `pages/VideoPlayer`，Hint 包含 `视频播放器` / `视频设置`
- 本地目录 → 本地列表：目标页面 `pages/LocalVideoList`，Hint 包含 `视频列表`
- DLNA 拉起 → DLNA 播放页：目标页面 `pages/DlnaPlayer`，Hint 包含 `播放` / `暂停`

同时对“返回键行为”做了特殊说明：一般页面返回首页；`pages/DlnaPlayer` 属于特殊用例。

### 3.2 页面集成契约（启动方式/参数/返回栈）

对应文件：`entry/src/ohosTest/ets/test/PageIntegrationContract.test.ets`

主要覆盖：

- 首页（Index）入口的多种启动参数解析与导航：
  - 从 Launcher 正常启动

- 播放器页面参数契约：

## 4. 功能测试项

### 4.1 启动与基础交互

**目的**：验证应用可正常启动、页面展示与焦点导航正常。

**步骤**：

1. 启动应用。
2. 在主页使用遥控器方向键/键盘方向键移动焦点。
3. 分别聚焦「网络视频」「本地视频」入口，确认焦点样式与移动逻辑正确。
4. 按确认键/Enter 进入对应页面。

**预期结果**：

- 应用启动无异常退出
- 焦点可在可交互控件间移动
- 按确认键可进入目标页面

### 4.2 网络视频列表与播放

**目的**：验证网络视频列表展示、进入播放页与播放控制。

**步骤**：

1. 在主页进入「网络视频」。
2. 等待网络视频列表加载完成。
3. 选择任意视频条目进入播放器页。
4. 在播放器页进行以下操作：
   - 播放/暂停切换
   - 进度拖动/快进快退（若页面支持）
   - 退出播放返回列表/主页

**预期结果**：

- 列表可正常展示（网络可用时）
- 点击条目可进入播放页
- 播放过程音视频同步，控制操作响应正常
- 退出播放可回到上一级页面

### 4.3 本地视频浏览与播放

**目的**：验证本地目录浏览、视频列表展示与本地播放。

**步骤**：

1. 在设备中准备本地视频文件（确保应用具备读取媒体权限）。
2. 在主页进入「本地视频」。
3. 选择存储设备/目录，进入视频列表。
4. 选择任意视频进入播放页并播放。

**预期结果**：

- 可按目录浏览本地文件
- 可正确识别并展示视频文件
- 选择视频可进入播放页并正常播放

### 4.4 文件管理器拉起播放（viewData）

**目的**：验证通过文件管理器选择视频文件后可直接拉起播放器进行播放。

**前提条件**：

- 设备具备文件管理器应用
- 文件管理器可对视频文件发起 `ohos.want.action.viewData`

**步骤**：

1. 打开文件管理器，定位到任意视频文件。
2. 使用“打开方式/打开”选择本应用（或直接触发系统默认打开）。
3. 观察是否进入播放器页并开始播放。

**预期结果**：

- 应用被正确拉起
- 播放器页展示正常，可播放选中的视频文件

### 4.5 DLNA 播放与分布式流转（可选）

**目的**：在具备条件的环境下验证 DLNA 与跨设备流转。

**前提条件**：

- 多设备处于同一网络环境
- 系统/设备支持 DLNA 与分布式能力
- 权限与系统配置满足要求

**步骤（示例）**：

1. 启动应用并进入播放页。
2. 按照页面提供的 DLNA/流转入口进行设备发现与连接。
3. 选择目标设备进行投送/流转。
4. 在目标设备上确认播放状态与控制响应。

**预期结果**：

- 可发现并选择目标设备
- 投送/流转过程可用，播放状态符合预期
- 断开/退出后可恢复本机播放或返回上一级页面

## 5. 权限与注意事项

- 网络视频功能需要网络相关权限（如 `ohos.permission.INTERNET`）与可用网络
- 本地视频浏览/播放需要媒体读取相关权限（如 `READ_MEDIA`、`READ_IMAGEVIDEO` 等）
- 分布式与 DLNA 相关能力依赖系统权限与设备能力，可能需要特定签名/权限等级支持

如遇到列表空、无法播放、无法发现设备等问题，建议优先确认：网络连通性、权限授予、设备系统能力与配置。
